VulnerabilityOWASP A05 Security Misconfiguration — Sensitive data in HTML comments

// Mission Brief

Intelligence reports indicate that Atlas Core Systems deployed their internal staff portal to a live server without removing developer debugging notes. A careless developer left sensitive information somewhere in the page — but the rendered page looks completely normal.

Your mission: inspect the page source and recover the first flag.

// Hints

Hint 1 (click to reveal)

Have you tried looking at what the browser receives but does not display on the screen?

Hint 2 (click to reveal)

Right-click on the page and select "View Page Source." Search for the word "flag" or look for HTML comment tags.

// Blue Team — Vulnerability Explanation

Root cause: Sensitive debugging information was placed inside HTML comments in the production page source. HTML comments are delivered to every browser that loads the page.

Impact: Any user can press Ctrl+U or use DevTools to read HTML comments, exposing flags, API keys, credentials, internal paths, or debug notes.

Mitigations:

  • Remove all comments and debug notes before deployment.
  • Never store secrets, credentials, or flags in client-side code.
  • Use a code review checklist that includes comment inspection.
  • Store secrets server-side using environment variables or a secrets manager.
  • Never assume anything sent to the browser is hidden from the user.

OWASP Category: A05:2021 — Security Misconfiguration / Sensitive Data Exposure
