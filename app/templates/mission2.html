// Solve Steps (Python reference)

import base64

payload = "{{ payload }}"

# Step 1: Base64 decode
step1 = base64.b64decode(payload).decode()
print("After Base64:", step1)

# Step 2: Reverse the string
step2 = step1[::-1]
print("After reverse:", step2)

# Step 3: Caesar shift -3 (each letter shifted back by 3)
def caesar(text, shift):
    result = []
    for ch in text:
        if ch.isalpha():
            base = ord('A') if ch.isupper() else ord('a')
            result.append(chr((ord(ch) - base + shift) % 26 + base))
        else:
            result.append(ch)
    return ''.join(result)

flag = caesar(step2, -3)
print("Flag:", flag)

// Blue Team — Vulnerability Explanation

Root cause: Sensitive data was obfuscated using reversible encoding transformations (Base64, string reversal, Caesar cipher) instead of real cryptographic encryption. Any of these steps can be reversed by an attacker using free tools.

Impact: An attacker who recognizes the encoding format can decode the message using CyberChef or Python in minutes, without any key or secret.

Key distinction:

  • Encoding (Base64, Hex, URL encoding) is reversible transformation — it is not secret and provides no confidentiality.
  • Encryption (AES, RSA, ChaCha20) requires a secret key. Without the key, reversal is computationally infeasible.

Mitigations:

  • Use real cryptographic encryption (AES-256-GCM, RSA-OAEP) for sensitive data in transit.
  • Manage and protect encryption keys securely.
  • Never rely on encoding, obfuscation, or "security by obscurity" as a protection mechanism.
  • Follow OWASP A02:2021 — Cryptographic Failures guidance.
