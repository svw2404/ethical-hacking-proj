// Vulnerable Code (for learning)

# VULNERABLE: backend trusts the URL parameter without authorization check
@app.route("/profile")
def profile():
    profile_id = request.args.get("id")   # taken directly from URL
    user = db.execute(
        "SELECT * FROM profiles WHERE id = ?", (profile_id,)
    ).fetchone()
    return render_template("profile.html", profile=user)

# No check: "Is the logged-in user allowed to view profile_id?"
# Attacker changes ?id=102 to ?id=1 → admin profile exposed

// Blue Team — Vulnerability Explanation

Vulnerability: IDOR — Insecure Direct Object Reference / Broken Access Control (OWASP A01:2021)

Root cause: The server returns profile data based solely on the id parameter in the URL without verifying that the currently authenticated user has permission to view that profile.

Impact: Any authenticated user can view any other user's profile, including administrator records containing sensitive data or flags, by simply changing a number in the URL.

Mitigations — see Secure Version:

  • Always enforce authorization server-side: verify that session["user_id"] == requested_id before returning data.
  • Never trust URL parameters to determine what data a user can access.
  • Use indirect object references (e.g., UUIDs instead of sequential IDs).
  • Log unauthorized access attempts for audit and alerting.

OWASP Category: A01:2021 — Broken Access Control
